Proton-io Module

Proton-io module for Easy Discord.JS V12 command handling. This not module. Much more for handling!

Proton modül kolay kullanımlı discord.js v12 komut yükleyici fakat bir komut yükleyiciden çok daha fazlası!

Venosa Npm sayfası GitHub'da gör

npm install proton-io

Proton modül rehberine hoş geldiniz!

Proton Modül rehberimizin başında hatırlatmak istediğimiz bazı hususlar bulunmaktadır. Proton modül öğretici rehberinin tümünde kullanıcıların Node.JS programlama dilinde bilgi sahibi olduğu kabul edilir.

Index.JS dosyamızda protonu entegre etme!

Bu bölümümüzde basit bir şekilde proton modülümüzü indireceğiz ve daha sonrasında index.js dosya yapısının nasıl olması gerektiğini öğreneceğiz.


İlk olarak botumuzun bulunduğu klasörde sol shift tuşuna basılı tutarak fareniz ile sağ tıklayınız.

Açılan seçenekler arasından komut istemini burda aç veya Powershelli burda aç seçeneğine tıklayınız.

Açılan panele şu kodu giriniz: npm install proton-io (Bu işlemde Node.JS yüklü sayılır.)

Proton-io ve Discord.JS modüllerinin ikisinide yüklüyoruz ve daha sonrasında bot.js veya index.js'de bunu belirtiyoruz.

Örnek alt kısımda bulunmaktadır: 


const Discord = require("discord.js") //Discord.JS tanımladık.
const Proton = require("proton-io") //Proton modülümüzü tanımladık.

Örnek index.js kısmı:


const Discord = require("discord.js") //Discord.JS tanımladık.
const Proton = require("proton-io") //Proton modülümüzü tanımladık.
const client = new Discord.Client(); //Botu burda tanımlıyoruz.

//proton ayarlar kısmımız bu kısımda
let komutKlasör = "./komutlar" //komutların olacağı klasörü seçiniz. Başındaki ./ kısmını kaldırmayınız.
let eventLoad = "./events" //Eventlar discordda bir olay olduğunda çalışan genel öğelerdir yine bunları ayrı bir dosya olarak alabiliyoruz. Event klasörünüzü değiştirebilirsiniz. Başındaki ./ kısmı kaldırmayınız. 
let ownerIDs = ["576117387645026314", "276057718354083860"] // Sahip kısmı iki kişiye kadar destek verir. Bu kısıma geliştirici ID'lerini giriyoruz ve modülde hazır bulunan geliştirici modüllerini bu iki geliştirici kullanabiliyor.
let owner = true; //Burda geliştirici komutlarının açık mı kapalı mı olacağını yazıyoruz. 
let defaultCommands = ["ping"]; //Burda modülün kendisinden gelen ping komutunu kullanmak istiyorsanız bu şekilde kullanmak istemiyorsanız [] bu şekilde bırakınız.

//burda ayarlar kısmında belirttiğimiz ayarlarımızı proton modülümüze aktarıyoruz.
//ilk parametrede client'i aktarıyoruz
//ikinci parametrede komutlar klasörümüzü aktarıyoruz.
//üçüncü parametrede ownerlar yani sahipleri aktarıyoruz.
//dördüncü parametrede owner ve defaultCommands verilerini aktarıyoruz.
const komutYukle = new proton(client,komutKlasör,eventLoad,ownerIDs,{owner,defaultCommands})

//Proton modülü aynı zamanda mesajlardada bazı ayarlar ve imkanlar size sunuyor bu ayarlarımızıda burda kullanalım:
let botlaraCevapVer = false;//adındanda anlaşılacağı gibi bu özelliği false yaptığınızda hiçbir botun komutuna tepki vermez.
let etiketiPrefixOlarakKullan = true; // etiketlediğinizde botunuz prefix olarak kabul eder ve çalışmaya devam eder.
let etiketlePrefixOgren = true; // etiketlediğinizde botunuz otomatik olarak prefixini söyler.

//discord.js message eventını çağırıyoruz ve burdan gerekli ayarları proton sayesinde entegre ediyoruz.
client.on('message', async (msg) => {
  komutYukle.message(msg,prefix,{botlaraCevapVer,etiketiPrefixOlarakKullan,etiketlePrefixOgren})
})


client.login("TOKEN")//Botumuzu giriş yaptırıyoruz.
//Token: discord.com/developers kısmındaki token bilginiz.
